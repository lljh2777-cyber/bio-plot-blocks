# BioPlotBlocks 可视化绘图模式改版说明

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 改动名称 | BioPlotBlocks 可视化绘图模式改版 |
| 文档类型 | 产品设计与开发说明 |
| 版本 | v0.1 |
| 状态 | 计划开发 |
| 更新日期 | 2026-07-12 |

## 2. 改动背景

BioPlotBlocks 的初衷是让科研绘图更可视化、更直观，并降低 R 和 ggplot2 的使用门槛。

当前软件已经具备模块库、图层栈、原生参数检查器、R 代码生成和本地预览等能力，但默认交互仍然以 ggplot2 的函数和参数为中心。用户即使只想创建一张简单散点图，也需要理解以下概念：

- `ggplot()`；
- `aes()` 和 mapping；
- geom 与 layer；
- 映射属性与固定属性的区别；
- `Unset`、`NULL`、Raw expression 等参数状态。

因此，当前产品更接近“可视化的 ggplot2 代码编辑器”，而不是面向非 R 用户的“可视化绘图工具”。

## 3. 核心问题

当前界面以“R 函数和参数”作为用户的主要心智模型，但普通科研用户的真实任务是：

- 我想画一张散点图；
- 我想用分组列给点着色；
- 我想突出显著基因；
- 我想调整颜色、点大小和标题；
- 我想导出图片或可重复使用的 R 代码。

本次改版需要将默认交互从“函数驱动”调整为“绘图任务驱动”。

## 4. 改动目标

### 4.1 主要目标

- 不会 R 的用户也能独立完成常见科研绘图。
- 用户以“图表类型、数据字段和样式”而不是 R 函数作为主要操作对象。
- 常见图表在选择数据和字段后立即生成预览。
- 保留现有模块、语义状态和 R 代码生成引擎，避免重写底层。
- 同时满足初学者的易用性和 R 用户的精细控制需求。

### 4.2 非目标

- 本次改版不删除现有模块栈和原生 R 参数能力。
- 不要求第一阶段支持所有 ggplot2 图形和扩展包。
- 不在无法确定语义时静默修改用户自定义的 R 表达式。

## 5. 目标用户

### 5.1 主要用户

- 不会 R 或只会少量 R 语法的生物信息学和生命科学用户；
- 熟悉 Excel、GraphPad Prism 或其他桌面绘图软件的用户；
- 需要快速生成标准科研图表的学生、实验人员和分析人员。

### 5.2 次要用户

- 熟悉 R/ggplot2，但希望通过可视化操作加快绘图的用户；
- 需要检查、导出或继续编辑生成 R 代码的用户。

## 6. 产品设计原则

1. **先选任务，后显示参数**：先让用户选择图表类型，再显示与该图表有关的设置。
2. **默认即可用**：根据数据类型提供可执行的默认映射和样式。
3. **所见即所得**：修改字段或常用样式后快速刷新预览。
4. **渐进式复杂度**：常用设置直接展示，高级参数按需展开。
5. **R 代码是输出，不是前置知识**：普通用户无需阅读 R 代码才能完成绘图。
6. **不静默丢失语义**：不能安全转换的内容应保留并向用户说明。

## 7. 双模式设计

软件提供两种可切换的界面模式，并共享同一份项目状态。

### 7.1 可视化模式

可视化模式作为默认模式，面向不熟悉 R 的用户。

- 以图表类型和数据字段为中心；
- 不显示 `Unset`、`NULL`、Raw expression 等原生状态；
- 默认隐藏函数库、模块栈和生成的 R 代码；
- 提供中文任务名称、示例和即时提示；
- 使用软件自动推荐的字段、配色和主题。

### 7.2 R/高级模式

R/高级模式保留现有精细控制能力。

- 显示函数库和图层栈；
- 显示 ggplot2 原生参数和语义状态；
- 支持 Raw R expression；
- 显示并导出完整 R 代码；
- 允许精确调整图层顺序和函数参数。

### 7.3 模式切换规则

- 两种模式操作同一份项目模型；
- 切换模式不得丢失图层、映射、样式或自定义 R 表达式；
- 高级模式中无法用可视化控件表示的内容，在返回可视化模式时应以“高级设置已保留”标记呈现；
- 任何模式中的操作均应支持 Undo/Redo。

## 8. 默认绘图流程

打开软件后，默认显示以下流程：

```text
1. 选择或导入数据
          ↓
2. 选择图表类型
          ↓
3. 选择 X、Y、分组等数据字段
          ↓
4. 自动生成并刷新预览
          ↓
5. 调整样式、标题和标签
          ↓
6. 导出图片、项目或 R 代码
```

## 9. 首个完整样板：散点图向导

第一阶段应首先完成散点图的端到端可视化工作流，作为其他图表类型的实现样板。

### 9.1 入口

用户单击：

```text
创建图表 → 散点图
```

软件在底层自动创建：

```r
ggplot(data = selected_data, mapping = aes(x = selected_x, y = selected_y)) +
  geom_point()
```

可视化模式中不向用户暴露上述实现细节。

### 9.2 基本字段

```text
数据源：  [df                               ▼]
X 轴：     [log2FC                           ▼]
Y 轴：     [neg_log10_padj                   ▼]
按字段着色：[不使用                          ▼]
点大小字段：[不使用                          ▼]
标签字段：  [不使用                          ▼]
```

X 和 Y 是必选字段；其他字段可选。

### 9.3 常用样式

```text
统一点颜色：[颜色选择器]
点大小：      [1.6]
透明度：      [0.7]
点形状：      [圆形 ▼]
配色方案：    [默认分组色 ▼]
趋势线：      [关闭 / 线性 / 平滑]
```

### 9.4 标题和坐标轴

```text
图表标题：[                              ]
X 轴标题： [                              ]
Y 轴标题： [                              ]
图例标题：[                              ]
X 轴尺度： [线性 / log10]
Y 轴尺度： [线性 / log10 / -log10]
```

### 9.5 预览规则

- 选择有效的 X 和 Y 后自动生成第一张预览；
- 常用样式调整采用短延迟自动刷新；
- 耗时操作仍可使用“运行预览”按钮确认；
- 新设置不可执行时，保留上一张成功预览并显示可操作的错误信息。

## 10. 图表类型库

可视化模式中使用图表卡片，而不是 R 函数列表。

### 10.1 第一批支持

- 散点图；
- 火山图；
- 箱线图；
- 小提琴图；
- 柱状图；
- 折线图；
- PCA 图；
- 密度图。

### 10.2 图表卡片内容

每张卡片包含：

- 图表名称；
- 小型示例图；
- 适用数据类型；
- 必需字段；
- 常见用途；
- 数据不匹配时的不可用原因。

## 11. 面向普通用户的术语

可视化模式中应使用任务导向的名称。

| R/ggplot2 术语 | 可视化模式名称 |
|---|---|
| mapping / `aes()` | 数据字段 |
| geom | 图形元素 |
| layer | 图层或元素 |
| `aes(color = group)` | 按字段着色 |
| `color = "blue"` | 统一颜色 |
| `facet_wrap()` | 分面显示 |
| `scale_*()` | 坐标轴或配色设置 |
| `Unset` | 使用默认值 |
| Raw expression | 自定义 R 表达式 |
| `theme_*()` | 图表主题 |

### 11.1 映射和固定样式必须分开

界面不得要求普通用户理解 `aes()` 内外的区别，而应使用两类独立控件：

```text
按字段着色：[分组列 ▼]
统一点颜色：[颜色选择器]
```

如果启用“按字段着色”，统一颜色可显示为“由分组配色控制”，避免交互冲突。

## 12. 智能推荐

### 12.1 数据列识别

导入数据后自动识别：

- 数值列；
- 分类列；
- 日期和时间列；
- 高基数文本列；
- 基因或特征标识列；
- P 值和 FDR 候选列；
- fold change 候选列；
- 分组和状态列。

### 12.2 图表推荐

| 数据特征 | 推荐图表 |
|---|---|
| 两个数值列 | 散点图 |
| 一个数值列 + 一个分类列 | 箱线图、小提琴图 |
| logFC + PValue/FDR | 火山图 |
| PC1 + PC2 | PCA 图 |
| 时间列 + 数值列 | 折线图 |
| 分类列 + 计数或数值列 | 柱状图 |

### 12.3 字段推荐

- 当仅有两个合适的数值列时，可自动填充 X 和 Y；
- 存在低基数分类列时，可推荐作为颜色分组；
- 标识列可推荐作为标签，但默认不显示所有标签；
- 推荐只是可撤销的初始值，不覆盖用户明确选择。

## 13. 可视化数据处理

生物信息绘图中的常见转换不应强制用户预先在 R 中完成。

### 13.1 字段转换

```text
字段：[FDR ▼]
转换：[不转换 / log10 / -log10 / 绝对值]
```

### 13.2 筛选条件

```text
FDR < 0.05
|log2FC| > 1
regulated != "NS"
```

界面使用字段、操作符和值的组合控件，底层再生成可检查的 R 表达式。

### 13.3 分组创建

火山图向导可提供：

```text
上调：log2FC > 1 且 FDR < 0.05
下调：log2FC < -1 且 FDR < 0.05
其他：不显著
```

软件依据规则生成用于着色和图例的分组字段或表达式。

## 14. 错误和反馈设计

### 14.1 错误信息原则

错误必须回答三个问题：

1. 发生了什么；
2. 为什么发生；
3. 用户现在可以做什么。

### 14.2 列名缺失示例

```text
无法绘制图形

当前 Y 轴使用了“neg_log10_padj”，
但数据 DEG_edgeR_symbol 中没有这一列。

建议：
• 从 Y 轴列表中选择 FDR 或 PValue；
• 对 FDR 使用“-log10”转换；
• 切换回包含该列的数据源。
```

### 14.3 自定义 R 表达式

对于无法安全推断输出列的表达式，显示：

```text
已检测到自定义 R 数据表达式。
由于无法安全确定输出列，已保留当前字段设置。
请运行预览验证结果。
```

### 14.4 预览失败

- 保留上一张成功预览；
- 错误信息区支持滚动和复制；
- 高亮可能导致错误的字段或设置；
- 在高级模式中可同时查看原始 R 错误。

## 15. 项目和数据状态同步

可视化模式和 R 模式必须使用同一语义来源。

用户切换数据源后，软件自动：

- 设置活动数据源；
- 切换数据预览；
- 更新可选列名；
- 保留新数据中仍存在的映射；
- 清除不存在的直接列映射；
- 保留不能安全推断的自定义表达式；
- 更新生成的 R 代码；
- 记录 Undo/Redo 历史。

`relink required` 数据源不能成为活动绘图数据，且不得改变现有映射。

## 16. 底层架构复用

本次改版不建议重写当前的 R 语义和模块引擎。

### 16.1 推荐分层

```text
可视化任务层
图表类型、数据字段、样式、转换
              ↓
图表配置层
ScatterConfig / VolcanoConfig / BoxplotConfig
              ↓
现有模块与语义层
ggplot / aes / geom / scale / labs / theme
              ↓
代码生成与本地 R 执行
```

### 16.2 配置对象示例

```text
ScatterConfig
├─ data_source
├─ x_field
├─ y_field
├─ color_field
├─ size_field
├─ label_field
├─ point_color
├─ point_size
├─ alpha
├─ trend_line
├─ title
├─ x_label
├─ y_label
└─ theme
```

配置层负责将用户意图翻译为现有模块状态，而不直接拼接 R 字符串。

## 17. 分阶段实施计划

### 第一阶段：散点图可视化闭环

- 新增“可视化模式 / R 模式”切换；
- 新增“创建图表”入口；
- 完成散点图向导；
- 提供数据源、X、Y、着色、大小和标签选择；
- 提供基本标题、配色和主题设置；
- 实现字段类型识别和 X/Y 推荐；
- 保持与现有模块、预览和代码生成同步。

### 第二阶段：常用科研图表向导

- 火山图；
- 箱线图和小提琴图；
- PCA 图；
- 柱状图和折线图；
- 各图表的专用字段校验和智能默认值。

### 第三阶段：数据转换与筛选

- 可视化字段转换；
- 可视化筛选条件；
- 分组规则创建；
- 因子顺序和标签重命名；
- 转换步骤的 R 代码导出。

### 第四阶段：交互优化

- 拖放字段到 X、Y、颜色、大小等角色；
- 新用户引导和示例任务；
- 图表搜索和推荐排序；
- 常用样式预设；
- 更完整的可访问性和小屏布局。

## 18. 验收标准

### 18.1 核心可用性

- 完全不会 R 的用户可以独立完成散点图。
- 从选择数据到生成第一张散点图，不超过 5 次主要操作。
- 在数据已导入的情况下，用户应能在 1 分钟内生成散点图。
- 完成基本绘图不需要理解 `ggplot()`、`aes()`、geom、`Unset` 或 `NULL`。
- 80% 的常见图表设置可在不打开 R/高级模式的情况下完成。

### 18.2 数据与字段

- 数据源切换后，预览、字段候选和映射状态保持一致。
- 无效的直接列映射会被清除并显示明确提示。
- 可用的映射、固定样式和标题不得因数据切换被清空。
- 不可用数据源不得导致当前项目状态与界面显示不一致。

### 18.3 预览和代码

- 可视化设置与生成的 R/ggplot2 代码语义一致。
- 从可视化模式切换到 R 模式后，可见对应的普通模块和原生参数。
- 返回可视化模式时，高级设置不得丢失。
- 所有可视化操作均可 Undo/Redo。
- 预览失败时不丢失上一张成功图像。

### 18.4 用户测试

至少邀请以下用户执行散点图任务：

- 3 名不会 R 的生命科学用户；
- 2 名有基础 R 经验的用户；
- 1 名熟悉 ggplot2 的用户。

记录：

- 任务完成率；
- 完成时间；
- 主要操作次数；
- 需要解释的 R 术语数量；
- 错误恢复能力；
- 用户对结果的信心。

## 19. 预期效果

改版后，BioPlotBlocks 的主要交互将从：

```text
添加 ggplot()
→ 添加 geom_point()
→ 编辑 aes()
→ 理解原生参数
→ 运行预览
```

转变为：

```text
选择散点图
→ 选择 X 和 Y
→ 自动生成预览
→ 调整颜色和样式
→ 导出结果
```

现有模块和 R 代码引擎继续作为底层语义基础，而可视化模式负责将用户的绘图意图转换为可执行、可检查、可导出的 ggplot2 项目。

最终目标是让用户无需先学会 R，也能快速创建正确、可重复且可继续编辑的科研图表。
